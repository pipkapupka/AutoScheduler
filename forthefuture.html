Как пользоваться API учебного портала УКРТБ

Данный сервис разработан специально для студентов колледжа. Если у Вас появилось некое желание разработать какую-либо программу или приложение для нашего колледжа, то мы с радостью предоставим Вам информацию с учебного портала УКРТБ. Данный сервис в разы ускоряет работу всех, уже написанных приложений, т. к. пропадает необходимость «парсить» информацию со страниц нашего портала.

Для начала работы с API, необходимо обратиться по адресу:

https://api.ukrtb.ru/api/ <название метода>

Коды состояний API учебного портала

В ходе работы с нашим сервисом Вы столкнетесь с некоторыми состояниями:

  • 200 (OK) – запрос к ресурсу выполнен успешно;
  • 204 (No Content) – отсутствует содержимое, попробуйте изменить параметры;
  • 400 (Bad Request) – Плохой запрос. Скорее всего Вы забыли указать обязательный параметр запроса;
  • 401 (Unauthorized) – Не авторизован. Для использования сервиса необходимо с каждым методом передавать APIKey через заголовок. Также в некоторых методах, которые затрагиваю личные данные пользователей необходима авторизация в виде логина и пароля;
  • 403 (Forbidden) – Доступ запрещен. Скорее всего Вы пытаетесь обратиться к несуществующему методу.

Для удобства сформирована Postman-коллекция


Авторизация по APIKey

Для использования API учебного портала необходимо получить APIKey. При каждом обращении его необходимо отправлять в виде заголовка:

headers: {
    apikey: «apikey предоставленный сервисом»
}

Авторизация пользователя

Авторизация пользователя осуществляется с помощью BearerToken, который необходимо отправить в виде заголовка:

headers: {
    Authorization: "Bearer «BearerToken полученный с помощью метода auth»"
}

Методы обращения к API

Для работы с API было реализовано 10 методов обращения, для некоторых необходима аутентификация пользователя.


Метод getGroups GET

Метод позволяет получить полный список учебных групп, которые обучаются в текущем учебном году. Метод не имеет параметров.

В результате метод возвращает строку в формате JSON:

[{
    "group": "11ПБ-21к-22" // Наименование группы
}]

Метод getTeachers GET

Метод позволяет получить полный список преподавателей, которые преподают в текущем учебном году. Метод не имеет параметров.

В результате метод возвращает строку в формате JSON:

[{
    "fio": "Сафаров Владислав Маратович", // ФИО преподавателя
    "zoom": "https://us04web.zoom.us/j/575123370?pwd=RlFoOX..." // Ссылка на конференцию
}]

Метод getCabs GET

Метод позволяет получить полный список кабинетов, которые задействованы в текущем учебном году. Метод не имеет параметров.

В результате метод возвращает строку в формате JSON:

[{
    "cab": "102-Д", // Номер кабинета
    "name": "Кабинет истории и обществознания" // Название кабинета
}]

Метод getSchedule GET

Метод позволяет получить расписание занятий учебной группы, преподавателя и кабинета одновременно и отдельно.

Ожидаемые параметры:

{
    group: '9ИСП-31-20' // Наименование группы (необязательно)
    teacher: 'Сафаров Владислав Маратович' // ФИО преподавателя (необязательно)
    cab: '303' // Номер кабинета (необязательно)
    date: '2022-11-01' // Дата расписания занятий
}

В результате метод возвращает строку в формате JSON:

 
[{
    "num": 3, // Порядковый номер занятия
    "typeLesson": "Лекции", // Тип нагрузки
    "date": "2022-11-01", // Дата проведения
    "group": "9ОИБ-24К-21", // Наименование группы
    "lesson": "Вычислительная техника", // Наименование дисциплины
    "teacher": "Сафаров Владислав Маратович", // ФИО преподавателя
    "cab": "303", // Номер кабинета
    "dist": 0, // Дистанционное обучение
    "zoom": "https://us04web.zoom.us/j/2098182774?pwd=QV...",
    "time": {
        "start": "11:00", // Время начала занятия
        "end": "12:20" // Время окончания занятия
    }
}]
Метод auth POST

Метод позволяет авторизовать пользователя. Обратно возвращает BearerToken.

Ожидаемые параметры:

{
    "login": "login" // Логин пользователя
    "password": "password" // Пароль
}

В результате метод возвращает строку в формате JSON:

 
{
    "token": "fVtQhilxKmM..." // BearerToken
}
Метод getUser GET Аутентификация

Метод позволяет получить всю информацию о авторизованном пользователе.

В результате метод возвращает строку в формате JSON:

{
    "fio": "Фамилия Имя Отчество", // ФИО студента
    "role": 1, // Роль пользователя (1 - студент, 2 - Родитель)
    "group": "9ИСП-31-20", // Учебная группы
    "birthday": "2004-09-21", // Дата рождения
    "specialty": "Информационные системы и программирование" // Специальность
}
Метод getProgressType GET Аутентификация

Метод позволяет получить типы оценок, которые имеются у текущего пользователя.

В результате метод возвращает строку в формате JSON:

{
    "typeBall": "Дифференцированный зачет за семестр" // Тип оценок
}
Метод getProgress GET Аутентификация

Метод позволяет получить оценки студента за весь его учебный период:

В метод передаются параметры:

В результате метод возвращает строку в формате JSON:

Метод позволяет получить данные о посещаемости студента.

Ожидаемые параметры:

{
    "period": "1 семестр 2021/2022" // Учебный период

[{

Метод setReference POST

Метод позволяет отправить запрос на получение какой-либо справки в колледже. Возможно заказать 3 типа справок: справка об обучении в колледже, архивная справка, справка о размере стипендии.

Далее в зависимоти от выбранного типа справки задается набор параметров:

1 - Справка об обучении в колледже: 
{
    "type": 1, // Тип справки
    "name": "Фамилия Имя Отчество", // ФИО студента
    "demand": "Военкомат", // Место требования
    "phone": "8800555...", // Номер телефона
    "count": 1, // Количество
    "email": "mail@mail.ru" // Электронная почта студента (если не обходимо получить справку на электронный адрес, иначе оставьте параметр пустым).
}
2 - Архивная справка: 
{
    "type": 1, // Тип справки
    "name": "Фамилия Имя Отчество", // ФИО студента
    "phone": "8800555...", // Номер телефона
    "eduStart": "2010", // Год начала обучения
    "eduEnd": "2014", // Год окончания обучения
    "email": "mail@mail.ru" // Электронная почта студента (если не обходимо получить справку на электронный адрес, иначе оставьте параметр пустым).
}
3 - Справка о размере стипендии: 
{
    "type": 1, // Тип справки
    "name": "Фамилия Имя Отчество", // ФИО студента
    "phone": "8800555...", // Номер телефона
    "count": 1, // Количество
    "scholarshipDateStart": "2020-11-26", // Период начала получения стипендии
    "scholarshipDateEnd": "2020-12-26", // Период окончания получения стипендии
    "email": "mail@mail.ru" // Электронная почта студента (если не обходимо получить справку на электронный адрес, иначе оставьте параметр пустым).
}
